This manual explains how to use the RAW MAT Allocation Engine (offline browser tool). Choose your language above.

Step 1 — Import Excel files

  • Upload or paste three files: Excel1 schedule, Excel2 demand (COIS), Excel3 BOM/stock.
  • Column mapping is fixed by business rules (see table below).
  • Use Sample data to try the tool without real files.

Step 2 — Mapping & run allocation

  • Review the fixed mapping table and detected headers.
  • Unit whitelist: when mother unit ≠ child unit (non-SHT), add conversion ratios (default M↔YD).
  • SHT size-suffix table: when mother unit is SHT, read size suffix from Excel3.F mother name (e.g. 110x200cm). Add or edit rows as needed.
  • Click Run allocation. Missing rules or unit mismatches will block with an error message.

Step 3 — Filter, preview & export

  • Filter by SO, cutting date range, or mother material. Preview updates live.
  • Checkboxes can hide zero-demand SOs or rows.
  • Export CSV or Excel. Export uses the same filters as preview.
  • If inputs change after a run, re-run allocation before exporting.

Column Y (allocated flag)

For each sales order + mother material group: Y appears on all rows only when every child has demand > 0 and provided ≥ demand (fully allocated). If any child is short, the whole group shows blank.

Troubleshooting

  • Unit pre-check failed: same mother must have matching Excel2.R and Excel3.H.
  • Missing conversion: add unit or SHT size-suffix rule in Step 2, then re-run.
  • SHT mother missing suffix: ensure Excel3.F mother name ends with size like 110x200cm.
  • Results stale: inputs changed — run allocation again before export.
Data stays on your computer. Do not upload real business files to external services.
