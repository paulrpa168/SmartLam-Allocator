This manual explains RAW MAT Allocation Engine v3 (offline browser tool). Choose your language above.

App version: allocation-web.html v3.1.6 — four inputs, 18-column export (mother plant/outside display), Segment FLT/MTF lock; MH04 kept, ignored for Y kit. Keep vendor/xlsx.full.min.js beside the HTML for offline Excel import.

Step 1 — Import Excel files

  • Upload or paste four files: Schedule, COOIS (demand), ZRMM0028 (BOM/outside), MB52 (stock).
  • Columns are resolved by English header names (aliases allowed). Dual-header sheets (Chinese then English) use the English row.
  • Use Sample data to try the tool without real files.
  • Import validation must show OK on all four slots before you can run allocation.
  • Offline Excel: open allocation-web.html from the project folder that also contains vendor/xlsx.full.min.js (do not copy the HTML alone).

Step 2 — Mapping & run allocation

  • Review the mapping table and detected headers for all four files.
  • Unit whitelist defaults (editable): M→YD=1.0936, YD→M=0.9144. Same units always 1:1. Use Reset defaults if the table is empty.
  • SHT size-suffix defaults (editable): 110x200cm / 220x110cm / 200x110cm. Suffix comes from ZRMM Material Full Description.
  • Click Run allocation. Missing rules or unit mismatches will block with an error message.

Allocation rules (important)

  • Demand path: only materials that exist as ZRMM0028 Article (child) are allocated. COOIS lines for HC/HU/etc. not in 0028.F are skipped.
  • MH04* children are kept in BOM / outside / demand / export. They are ignored only when deciding pending allocation (Y) kit completeness.
  • Outside stock = max(0, Σ GI J − Σ GR P) per mother–child–Batch (rows with J=0 or L=0 skipped for outside only). Mother outside column sums those pairs for mother+Batch. BOM expand keeps all non-39* children even when J/L is 0.
  • Plant stock = SUM(MB52 Unrestricted) by Material + Stock Segment (exclude storage 39*). Allocatable pool = child MB52 only. Mother plant stock is display-only from the same MB52 rule.
  • Segment lock: COOIS Requirement Segment = MB52 Stock Segment = ZRMM Batch. FLT and MTF never mix.
  • Mother COOIS lines expand to all valid children (× conversion); child COOIS Open Quantity adds as demand direct when the child is in 0028.F.
  • Allocate greedily by earliest cutting, then SO; mother rows sort above blank-mother direct rows within the same SO.

Step 3 — Filter, preview & export

  • Filter by SO, cutting date range, or mother/child material. Preview updates live.
  • Hide zero-demand SO / row checkboxes are ON by default. Uncheck to include demand=0 rows in preview and export.
  • Export CSV or Excel (18 columns). Excel export paints dual-identity material codes in blue (mother/child columns).
  • Blue codes = dual identity (same code is both mother and child in ZRMM0028).
  • If inputs change after a run, re-run allocation before exporting.

Column Y (pending allocation)

For each sales order + mother + segment group: Y when every non-MH04 demand>0 child is fully provided. MH04 children do not affect the kit flag. Direct-only rows (blank mother) form their own group per child.

Output columns (18)

Preview and Excel use the same column order.

  1. cutting · so
  2. mother material · mother batch · mother unit
  3. mother plant stock · mother stock outside (display only)
  4. child material · child batch · child unit
  5. child demand · demand direct · demand qty
  6. child stock (running) · stock available (=MB52 pool)
  7. provided qty · remaining stock after this row · pending allocation (Y)

Troubleshooting

  • Unit pre-check failed: same mother must have matching COOIS Base Unit and ZRMM OUn.
  • Missing conversion: add unit or SHT size-suffix rule in Step 2 (or Reset defaults), then re-run.
  • SHT mother missing suffix: ensure ZRMM description ends with size like 110x200cm.
  • Results stale: inputs changed — run allocation again before export.
  • HC/HU rows missing: only materials present as ZRMM Article (after J/L and 39* filters) are calculated — by design.
  • If available stock looks wrong: re-check Segment lock (FLT/MTF) and that MB52 Unrestricted rows are not on storage 39*.
Data stays on your computer. Do not upload real business files to external services.
